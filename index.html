Interface Controller

The Rover Interface Controller (Rover IFC) is a micro service that integrates different integrations for StayNTouch PMS using the Pub/Sub model. The project involves listening to various events generated by the PMS application and based on that event, using the RPC method the application will communicate to different vendors via HTTP, Socket, Email, SFTP, S3 and other mechanisms in different content type formats (XML, JSON, TEXT, CSV etc).

What I did:
  • Developed action publish model which communicates with multiple consumers.
  • Integrated various third party CRS, CRM, RMS and back office, door lock and text messaging systems.
  • Involved in various technical discussions with vendors prior to integration.
  • Designed, developed and documented various modules that handle integration data.
  • Involved in implementing RabbitMQ to exchange messages between the interface and data provider applications.
  • Developed Webhook application for event reporting to vendors via HTTP.
  • Integrated Point-of-Sale system with the property management system using the interface application.
  • Walked clients and internal staff through troubleshooting steps to resolve common software issues.
  • Worked closely with the QA Engineers to build the test scenarios.

Property Management System

StayNTouch PMS (Property Management System) is Hotel Enterprise PMS overlay and Mobility Platform for the hotel guests and staff. The scope of the project involves a reservation Check In/Checkout application with RestFul web services which can be used by the Rover and Zest applications. The Rover Application which was developed in Ruby on Rails platform, serves as the backend for guests and staff and also a connect engine to the PMS named such as Opera. PMS also connects to another application called IFC, which communicates with external vendors using XML, JSON. Another product of Stayntouch is Rover Connect, which provides 3rd party applications read & write access to hotel data within the Rover PMS. All 3rd party applications must register with the Rover API Portal. The OAuth 2.0 standard is used by Rover. Hotels must grant 3rd party applications access to their data.

What I did:

  • Worked closely with a team of engineers and product owner to design and develop full architecture of the hotel availability module.
  • Database design for iBeacon setup (for mobile and web apps)

  • Push notification implementation for iOS and Android (Back-End)
  • Designed and developed reservation cancellation module which is key part of PMS
  • Designed and developed a complete HouseKeeping module for Hotels.
  • Implemented the inventory design for PMS.
